## 1. 这个东西是什么

**签到管家**(目录名"签到脚本聚合管理面板",域名 `jb.aijiaxia.cc`)是一个**自托管的 Web 控制台**,把你日常零散的 Python 签到脚本(B 站、coklw、各种论坛、薅羊毛网站等)统一管起来:

- **不再手动跑** —— 服务器 7×24 按 cron 自动执行
- **不再到处找配置** —— 所有 cookie / token 在一个网页里改
- **不再漏掉失败** —— 失败 / 成功推送到你的 Telegram / 微信 / 钉钉
- **不再看一堆黑乎乎终端** —— 每次运行有日志、状态、耗时、统计图

简单说:**让你给"自动化"这件事本身不再花心力**。

### 为什么不直接用青龙面板?

| 维度 | 青龙 | 签到管家 |
|---|---|---|
| 语言 | Node.js 主体 | Python + React |
| 支持脚本 | JS/Python/Shell 全支持 | **只 Python**(够用就好) |
| 多用户 | 支持 | **单用户**(自用足够) |
| 脚本配置 | 全局环境变量 | **每脚本自描述 → 自动生成表单 UI** |
| UI 风格 | 上一代审美(2020-2022) | shadcn 现代 + 深浅色 + ⌘K 命令面板 |
| 加密 | 无字段级加密 | **Fernet 字段级加密**(主密钥独立文件) |
| 镜像 | 几百 MB | < 200 MB |
| 适合谁 | 撸羊毛社区 / 大量公共脚本 | **个人 / 小团队,在意 UI 与安全** |

一句话:**青龙是仓库,签到管家是精装公寓**。

## 2. 现有功能清单(已生产上线,版本 MVP-4)

### 🔐 账号与会话

- 首次访问引导设置管理员密码(`/setup`),之后跳登录页
- 用户名 + 密码登录(bcrypt 哈希存储),HttpOnly Cookie + CSRF Token 双保险
- 改密页面 `/settings/account`(支持改用户名 / 改密码)
- 单用户(本期不做多人,后续可加)

### 📋 脚本管理(`/scripts`)

- 列表卡片视图,每个脚本展示:图标 / 名称 / 描述 / 启用状态 / 实例数 / 最近运行
- **重新扫描**按钮 → 后端遍历 `scripts/` 目录,新脚本自动入库,旧的不动
- 点卡片进详情页,Tab 切换:
  - **概览**:manifest 元信息、依赖、版本、README
  - **实例**:这个脚本下你建的所有任务(下文说"实例")
  - **历史**:这个脚本所有运行记录
  - **实时日志**:实时跟随最新一次运行的 stdout/stderr(SSE 推送)

### 🎯 实例(同一脚本可多账号 / 多组配置)

- 每个脚本下可以建任意多个**实例**:比如你有两个 B 站账号,就建两个实例,各自填 cookie + cron + 通知规则
- 创建实例时**根据 manifest.yaml 字段定义自动生成表单**:
  - `string` → 文本框
  - `secret` → 密码框(提交后服务端 Fernet 加密落库,前端再也看不见明文)
  - `integer` → 数字输入(支持 min/max)
  - `boolean` → 开关
  - `select` / `multiselect` → 下拉单/多选
  - `multiline` → 多行文本
  - `cron` → cron 输入框 + 实时人话翻译("每天上午 9:00")+ 未来 5 次执行时间预览
  - `url` → URL 输入 + 格式校验
  - `json` → JSON 编辑框 + 语法校验
- **每实例独立 cron**(可与脚本默认不同)、**独立 timeout**(超时强杀)、**独立通知规则**
- 操作:**立即运行** / **暂停调度** / **编辑配置** / **删除实例**

### 🔁 调度引擎(后台默默跑)

- APScheduler 3.x AsyncIOScheduler,**单进程同步模式**(uvicorn `--workers 1` 强约束)
- 任务持久化到独立 SQLite(`data/scheduler.db`,与业务库分开)
- 应用重启后调度自动恢复(不丢任务)
- 错过执行(misfire)24h 内会补跑

### 🧪 沙箱执行(每次运行隔离)

每次实例触发,后端干这些事:

1. 解密该实例的所有 `secret` 字段
2. 用 `subprocess.Popen` 新开一个 Python 进程跑脚本(`start_new_session=True` 建独立进程组)
3. 配置以 JSON 从 stdin 传入(脚本读 `sys.stdin`)
4. 脚本写日志到 stdout/stderr,实时被父进程捕获
5. 脚本最后必须打印 `__RUN_RESULT__{...JSON...}`,父进程解析为结果对象
6. 超时(实例的 `timeout_sec`)到达,父进程 `os.killpg(SIGTERM)` → 5 秒等待 → `SIGKILL` 强杀整个进程组
7. 子进程**无法 `import app.core.crypto`**(用独立 `sandbox_runner.py` 入口,不在 `app/` 树下)→ 主密钥不会被脚本意外读到

### 📊 仪表盘(`/dashboard`,首页)

- **核心指标卡片**:总脚本数 / 启用实例数 / 今日运行 / 今日失败 / 24h 通知数
- **7 天 sparkline 趋势**(成功 / 失败 / 总数)
- **即将运行**(下 5 个最近 cron 触发时间 + 倒计时)
- **最近失败**(最近 10 条失败记录 + 错误摘要 + 跳详情)
- **运行时间线**(按小时 / 按天 bucket,可切换)

### 📜 全局运行历史(`/runs`)

- 所有脚本所有实例的运行记录,统一时间线
- 筛选:按脚本 / 按实例 / 按状态(成功 / 失败 / 跳过 / 取消 / 超时)/ 按时间范围
- 点单条进详情:完整 stdout / stderr / 用了多久 / 退出码 / 结果摘要

### 🔔 通知(`/notifications`)

- **渠道管理**:基于 apprise 添加任意推送目标(Telegram / 钉钉 / 飞书 / 微信 / Server酱 / Bark / Pushover / 邮箱 SMTP 等 80+ 种)
- **规则管理**:绑定"哪个实例发生哪个事件"→"用哪个渠道推送"
  - 事件:运行成功 / 运行失败 / 运行超时 / 运行取消
  - 可设静默时间窗(比如 23:00–8:00 不打扰)
  - 可设错误抖动抑制(同一错误 10 分钟内只推一次)
- **模板自定义**:支持 Jinja2 模板,可引用 `{{ run.summary }}` `{{ instance.name }}` 等

### ⚙️ 设置(`/settings/*`)

- **`/settings/account`**:改用户名 / 改密码
- **`/settings/system`**:脚本扫描周期、默认时区、SSE 心跳秒数
- **`/settings/theme`**:配色主题(react-colorful 选 OKLCH 色)、深浅色、字体大小
- **`/settings/backup`**:导出整库 zip / 从 zip 导入(运维兜底)

### 🌗 全局 UI 体验

- 深 / 浅色一等公民,系统主题跟随
- ⌘K(Mac)/ Ctrl+K(Win)打开命令面板:全局跳页 / 快捷操作
- 侧栏可折叠(点击品牌区"签到管家"切换),CSS Grid 布局不会遮挡主内容
- 路由错误兜底(`RouteErrorBoundary`),React 崩溃也是友好降级页

## 3. 写一个新签到脚本(开发规范)

### 3.1 目录结构

```
scripts/<your-slug>/        ← slug 是英文小写横线,如 bilibili-daily
├── manifest.yaml           ← 必需:元信息 + 字段定义
├── main.py                 ← 必需:入口,实现签到逻辑
├── requirements.txt        ← 可选:Python 依赖(脚本独立 venv)
├── icon.svg                ← 可选:展示在卡片上的图标
└── README.md               ← 可选:用户准备步骤说明
```

### 3.2 manifest.yaml 完整范例


### 3.3 main.py 必须遵守的协议

```python
import json
import sys
import os
from datetime import datetime

def main():
    # 1. 从 stdin 读 JSON 配置(secret 已解密成明文)
    raw = sys.stdin.read()
    config = json.loads(raw)

    cookie_sessdata = config["sessdata"]
    cookie_bili_jct = config["bili_jct"]
    coin_count = config.get("coin_count", 1)

    # 可选:做随机延迟(避免每天固定时刻被风控)
    delay = config.get("random_delay_sec", 0)
    if delay > 0:
        import random, time
        actual = random.randint(0, delay)
        # 重要 sanity check:如果配置的 delay 大于剩余 timeout,提前 cap
        # (实例 timeout 通过 BACKEND_RUN_TIMEOUT_SEC 环境变量传入)
        timeout = int(os.environ.get("BACKEND_RUN_TIMEOUT_SEC", "300"))
        if actual > timeout - 60:
            actual = max(0, timeout - 60)
            print(f"[WARN] random_delay 被 cap 到 {actual}s(避免超时强杀)")
        time.sleep(actual)

    # 2. 你的签到逻辑
    try:
        result = do_signin(cookie_sessdata, cookie_bili_jct, coin_count)
        status = "success"
        summary = f"签到成功,获得 {result['exp']} 经验"
        metrics = {"exp": result["exp"], "coins": result["coins"]}
    except AlreadySigned:
        status = "skipped"
        summary = "今日已签到"
        metrics = {}
    except Exception as e:
        status = "failure"
        summary = f"签到失败: {e}"
        metrics = {}

    # 3. 关键:最后一行必须是 __RUN_RESULT__ + JSON
    print("__RUN_RESULT__" + json.dumps({
        "status": status,              # success / failure / skipped
        "summary": summary,            # 一句话给人看的结果
        "metrics": metrics,            # 任意业务指标,JSON 可序列化
    }, ensure_ascii=False))

    # 4. 退出码:0 = 成功 / 非 0 = 失败(failure 也用 1)
    sys.exit(0 if status != "failure" else 1)


if __name__ == "__main__":
    main()
```

### 3.4 11 种字段类型速查

| type | UI 表现 | DB 存储 | 适用 |
|---|---|---|---|
| `string` | 单行文本框 | 明文 | 用户名 / API 端点 |
| `secret` | 密码框 + 提交即加密 | Fernet 密文 | cookie / token / password |
| `integer` | 数字输入 + min/max | 整数 | 投币数 / 重试次数 / timeout |
| `boolean` | 开关 | 0/1 | 是否启用某功能 |
| `select` | 下拉单选 | 字符串 | 模式选择 |
| `multiselect` | 下拉多选 | JSON 数组 | 多账号 / 多目标 |
| `multiline` | 多行文本 | 字符串 | 长 cookie / 备注 |
| `cron` | cron 输入 + 人话预览 + 未来 5 次预览 | crontab 字符串 | 自定义调度 |
| `url` | URL 输入 + 校验 | 字符串 | webhook 端点 |
| `json` | JSON 编辑框 + 校验 | JSON 字符串 | 复杂结构化配置 |

### 3.6 ⚠️ 几条铁律

| 规则 | 原因 |
|---|---|
| **绝不 `import app.*`** | 脚本是沙箱进程,无权访问主程序内部 |
| **绝不写 `data/encryption.key`** | 主密钥与脚本物理隔离,你写不进去 |
| **`__RUN_RESULT__` 必须是最后一行** | 父进程倒着扫描 stdout 找 marker |
| **`timeout_sec` ≥ `random_delay_sec` + 60** | 否则随机延迟还没结束就被超时强杀 |
| **不依赖 `__pycache__`** | sandbox 用 `-B` 不写 pyc |
| **stdout / stderr 各最多 256 KiB** | 超出被截断,落库不会爆 |

## 5. 安全模型

| 资产 | 风险 | 措施 |
|---|---|---|
| 用户密码 | 撞库 / 弱密码 | bcrypt 哈希(cost 12),首次设置强度校验 |
| 会话 cookie | XSS 窃取 | HttpOnly + Secure + SameSite=Lax + CSRF Token |
| 字段 secret(cookie/token) | DB 泄露 | Fernet 加密,主密钥独立文件 |
| 主密钥 `encryption.key` | 泄露 = 全配置失效 | 600 权限 / 不入 git / 不入镜像 / 不入日志 / 必须离线备份 |
| 任意代码执行 | 用户上传恶意脚本 | **不做网页上传,只能 SSH 推目录**(物理隔离) |
| 子进程读主进程内存 | 拿到密钥 | 子进程不能 `import app.*`,主密钥不在 env |
| 长跑脚本卡死 | 资源耗尽 | `timeout_sec` 强杀,杀整个进程组 |
| API 横向越权 | 偷数据 | 全 API 强制 session 校验 + CSRF |
