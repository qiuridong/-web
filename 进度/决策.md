## 2026-05-16 · ADR-014 · 多 agent 并行编码必须做"契约对齐 review"(教训)

- **决定**:多 agent 并行写代码时,**PM 在派出 agent 前 + agent 回来后**都必须做 **API 契约对齐 review**:
  - **派出前**:检查每个 agent 的 prompt 里"输入字段假设"是否真的与设计稿 + 上游 agent 的产出对齐。特别警惕"沿用早期 mock 数据形状"(那很可能比真后端 schema 多字段)。
  - **回来后**:对每个跨 agent 接口画一张"字段归属表" — 列出后端真返字段 vs 前端组件引用字段,标 ✓(双方一致)/ ⚠(后端缺待补 / 前端假设错待修)/ ✗(完全冲突)。
  - 关键路径加防御性:前端任何 `.map(...) / .length` 操作前若数据来自后端,必须 `Array.isArray(x) ? x : []` 或 `x?.map(...) ?? []` 容错。
- **替代方案**:不做契约 review,让 bug 在用户报错后定位修
- **理由**:
  - 教训来源:**MVP-2 Dashboard 顶级 `.map(undefined)` 崩溃**(2026-05-16)。根因:6B(后端 dashboard schema)只返 § 2.7 设计稿最小集(5 字段),6C(前端 KpiCard)沿用 5B 时代 mock 引用了 `sparkline_7d_success` 等多余字段,真接通后 mock 字段不在 → `.map(undefined)` 崩
  - 三 agent 并行写代码本身没问题,但"看不见的接口契约不一致"是最常见 bug 来源,**必须主动检查**
  - 用户原话:"看起来之前问题是因为三者之间没有做好对接,之后你需要综合审核它们之间的计划是否合理"
- **代价 / 风险**:增加 PM 工作量(每批 agent 多 10-15 分钟 review);如果跳过会被用户 / 生产环境抓现行
- **回退路径**:N/A(这是流程规则,不是技术决策)

> 落实方式:派 agent 的 prompt **必须**带"上游 agent 已交付的接口签名"(URL / 返回字段全集);MVP-3 之后的 review 列在 `进度/分支/main.md` 的"集成 review"段。

## 2026-05-16 · ADR-013 · 部署目标确定(jb.aijiaxia.cc / 154.9.238.144 / Caddy 自动 HTTPS)

- **决定**:生产部署到 `jb.aijiaxia.cc`(A 记录 → `154.9.238.144`,Cloudflare 灰云直连),路径 `/opt/signin-panel`,SSH 用 root + `J:\密钥\美国质量8-8\vcs-deploy-rsa`
- **替代方案**:Cloudflare 橙云(走 CDN)/ 别的反代 + Certbot / 别的服务器
- **理由**:
  - **灰云直连**让 Caddy 的 HTTP-01 ACME challenge 能直接到源站,自动 HTTPS 一键完成,不必折腾 DNS-01 + Cloudflare API
  - 解析已就绪,SSH 密钥已准备好,可以随时进入"批次 6 集成验证"环节
  - 域名是子域(`jb.` 前缀),不影响主站
- **代价 / 风险**:
  - 源站 IP 暴露(没 CDN 防护)— 个人/小团队用,**可接受**;以后如要防 CC 攻击,可以打开橙云用 Cloudflare Tunnel 或改 DNS-01
  - root 直连 SSH 是基线方便,生产化建议后续加非 root deploy 用户 + 禁 root 登录
- **回退路径**:如要换域名/服务器,只需改 `.env` 里的 `DOMAIN` + `ACME_EMAIL`,Caddy 自动重申证书,代码零改动

## 2026-05-15 · ADR-012 · UI 库选 shadcn/ui + Tailwind v4(替换 Element Plus)

- **决定**:UI 基底用 **shadcn/ui**(Radix UI + Tailwind CSS v4 + lucide-react),数据可视化用 **Recharts** + 借鉴 **Tremor**,动画 **Framer Motion 12**;命令面板 **cmdk**;toast **sonner**;表单 **react-hook-form + zod**
- **替代方案**:Ant Design Pro / Mantine / Chakra UI v3 / NextUI(HeroUI)
- **理由**:
  - shadcn/ui 不是"装组件库",而是 copy-paste 源码到本仓库,**完全可定制**且不会随版本升级被破坏 — 是 React 生态美化天花板的代名词,Linear/Vercel/cal.com 同款基底
  - 配套 Radix UI(无样式 + 完整无障碍)+ Tailwind v4(原生 CSS 引擎、`@theme` directive)
  - 用户硬要求 "UI 必须美化",shadcn + Tailwind 给设计师/编码 agent 留足美化空间(对比 Element Plus 默认外观就是"不美化是默认状态")
  - 后续编码阶段配合 Claude Preview MCP 跑 dev server 实时验证,迭代效率高
- **代价 / 风险**:
  - 中文资料比 Ant Design 少,需要看英文文档
  - shadcn 组件需要自己维护(在 git 里)而不是 npm 包升级
  - Tailwind v4 是 2025 末新版,生态适配仍在迁移中(但 shadcn 已支持)
- **回退路径**:Ant Design Pro 也是 React 生态成熟方案,换库等于推倒前端,但路由 / 状态 / API 层不变

## 2026-05-15 · ADR-011 · 前端框架切换为 React(替换 Vue 3)

- **决定**:前端用 **React 18 + Vite + TypeScript**,路由 **React Router 7**,状态 **Zustand + TanStack Query 5**
- **替代方案**:继续用 Vue 3 + Element Plus(原 ADR-002)
- **理由**:
  - 用户主动要求切换("换 React")
  - React 生态的 UI 美化方案(shadcn/ui + Tailwind)更成熟,与"必须美化"要求契合度更高
  - TanStack Query 对服务端数据缓存的处理比 Pinia + axios 手写更优
  - React Router 7 的 `loader` + `redirect()` 模式做鉴权 guard 比 Vue 的 `beforeEach` 更声明式
- **代价 / 风险**:
  - 推翻原 ADR-002 + 原前端 UI 设计稿(已归档为 `设计/前端UI设计-旧Vue版.md` 保留参考)
  - 旧 Vue 设计稿里的视觉风格 token(色板、字号、间距等)仍然有效,新版 UI 设计稿会沿用相同设计语言
- **回退路径**:旧 Vue 稿仍在 `进度/设计/前端UI设计-旧Vue版.md`,如要切回直接复用;前后端契约(REST API + SSE)不变,前端框架切换不影响后端

> ⚠️ 因此 ADR-002 已被本条 ADR-011 + ADR-012 取代,保留原文供历史溯源。

## 2026-05-15 · ADR-001 · 后端语言选 Python

- **决定**:后端用 Python 3.11 + FastAPI
- **替代方案**:Node.js (NestJS)、Go (Gin)
- **理由**:用户的签到脚本几乎都是 Python,同语言可直接 spawn `python -m sandbox_runner`,接口契约统一为 Python 字典;FastAPI 异步性能好,OpenAPI 自动生成
- **代价**:Docker 镜像比 Go 大(~250MB vs ~20MB)
- **回退路径**:若以后 Go 重写,API 契约不变,前端不动

## 2026-05-15 · ADR-002 · 前端框架 Vue 3 + Element Plus(重度美化)

- **决定**:Vue 3 + Vite + TS + Pinia + Vue Router 4 + Element Plus
- **替代方案**:React + Ant Design Pro / shadcn/ui
- **理由**:中文生态最完整;Element Plus 组件齐全;有 vue-element-plus-admin 可参考布局;TS 支持成熟
- **代价**:Element Plus 默认外观土,**必须重度美化(SCSS 覆盖 CSS vars)**,不能用 default theme 上线
- **回退路径**:换 React 等于推倒重来,**重视决策**

## 2026-05-15 · ADR-003 · 调度器 APScheduler 3.x(不用 4.x)

- **决定**:APScheduler 3.11.x,AsyncIOScheduler 同 FastAPI 进程
- **替代方案**:APScheduler 4.0、Celery、独立 cron worker
- **理由**:4.x 仍 alpha(2026-05),作者明确警告不可生产用;Celery 对个人项目过度复杂;同进程 AsyncIO 共享 event loop、共享 DB session 简单
- **代价**:uvicorn 必须 `--workers 1`(否则多进程重复触发任务);进程重启 = 调度器重启
- **回退路径**:负载真上来再切独立 worker,不影响 API 契约

## 2026-05-15 · ADR-004 · 数据库 SQLite WAL(不上 PostgreSQL)

- **决定**:SQLite + WAL 模式,业务库 `data/db.sqlite3` 与 scheduler jobstore `data/scheduler.db` 分两个文件
- **替代方案**:PostgreSQL 容器
- **理由**:个人/小团队规模(实例 ≤ 1000,日 runs ≤ 10万),WAL 性能足够;零运维;备份就是复制单文件
- **代价**:不能多机部署;高并发写有锁(场景里几乎不会触发)
- **回退路径**:SQLAlchemy 模型不变,改 DATABASE_URL 即可上 PostgreSQL

## 2026-05-15 · ADR-005 · 脚本执行用子进程 + JSON over stdio

- **决定**:每次执行 `python -u -m runner.sandbox` 子进程,主进程通过 stdin 传 config+context JSON,子进程末行输出 `__RUN_RESULT__{...}` 标记结果
- **替代方案**:同进程 `import script.main; script.main.run(config)`
- **理由**:崩溃隔离、依赖冲突隔离、Python 线程无法强杀(超时熔断必须用进程组 SIGTERM/SIGKILL)、内存泄漏自然回收
- **代价**:每次启动 100-300ms 冷启动开销(签到场景可忽略);需实现 stdio 协议
- **回退路径**:manifest 加 `runtime.isolated: false` 选项允许 trusted 脚本同进程

## 2026-05-15 · ADR-006 · 加密用 cryptography.Fernet 字段级

- **决定**:`cryptography.Fernet`,主密钥 `data/encryption.key`(权限 600),`Instance.config_blob` 整个 JSON 加密落库;采用 MultiFernet 预留密钥轮换能力
- **替代方案**:整库加密(SQLCipher)、HashiCorp Vault
- **理由**:Fernet = AES-128-CBC + HMAC-SHA256,2026 仍是合理选择;字段级保留非加密字段的索引/查询能力;Vault 对个人项目过度
- **代价**:主密钥丢失 = 所有加密配置作废,**必须强警告 + UI 横幅提示备份**
- **回退路径**:MultiFernet 已支持轮换,后续可加换库流程

## 2026-05-15 · ADR-007 · 鉴权用 Server-side Session(不用 JWT)

- **决定**:bcrypt 密码哈希 + `secrets.token_urlsafe(48)` session token + HttpOnly + SameSite=Lax + Secure cookie;sessions 表存 SQLite
- **替代方案**:JWT、OAuth
- **理由**:JWT 撤销困难、localStorage 易被 XSS 拿;HttpOnly cookie 不可被 JS 读取;SameSite=Lax 防 CSRF;非 GET 请求额外要求 `X-Requested-With` 头双保险
- **代价**:服务端要存 session(无所谓,SQLite 单表)
- **回退路径**:若以后接 SSO,加一层 OIDC 适配,session 模型不变

## 2026-05-15 · ADR-008 · 反向代理用 Caddy(不用 Nginx)

- **决定**:Caddy 2 处理 HTTPS + 反代 + 静态前端
- **替代方案**:Nginx + Certbot
- **理由**:Caddy 自动 ACME 证书申请续期(零配置);反代 SSE 默认行为正确(Nginx 默认开缓冲会卡 SSE);Caddyfile 比 nginx.conf 友好太多;HTTP/3 默认开
- **代价**:内存占用比 Nginx 高 25MB;社区资源比 Nginx 少
- **回退路径**:换 Nginx 即可,只需重写 Caddyfile;`reverse_proxy` 与 `location` 等价转换简单

## 2026-05-15 · ADR-009 · OpenAPI → TS 类型用 openapi-typescript

- **决定**:`openapi-typescript` 生成 schema.d.ts,`openapi-fetch` 做类型安全 fetch
- **替代方案**:Hey API、Orval、手抄 interface
- **理由**:零运行时(6KB),官方维护,与 React Query 无关;Vue 项目最轻量
- **代价**:无 query cache(本项目用 Pinia store 兜住);需要 `pnpm gen:api` 手动触发
- **回退路径**:需要 cache 再加 @tanstack/vue-query

## 2026-05-15 · ADR-010 · 部署目标 Ubuntu 24 + Docker Compose

- **决定**:Docker Compose 三服务(caddy + backend + 前端 dist 由 caddy serve)
- **替代方案**:K8s、systemd 直接跑、PM2
- **理由**:个人服务器规模,Compose 一文件描述清晰;`docker compose up -d` 一键起;volumes 挂载 `data/` `scripts/` 持久化
- **代价**:无水平扩展;依赖 host 装 docker
- **回退路径**:负载真大才上 K8s,Compose → K8s 有迁移工具(kompose)
