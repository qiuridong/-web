## 2. 现状回顾:为什么 v2 还到不了 99%

### 2.1 v2 实际行为
当前 [`scripts/jmcomic/main.py`](../../scripts/jmcomic/main.py)(420 行)做的事:

```
1. 校验 cookie 字段(含 cf_clearance / remember / AVS 3 个 required)
2. 用 httpx 直接 POST https://18comic.vip/ajax/user_daily_sign
3. 解读 JSON:{"error": "finished"} = 已签 / {"msg": "..."} = 成功 / {"msg": ""} = 失效
4. 精细异常分 5 类(JmCloudflareBlocked / JmCookieExpired / JmSignEndpointError 等)
```

### 2.2 5 月数据(728 行 signin.log 实证)

| 日期 | 结果 | 原因(已证实) |
|---|---|---|
| 5-01 / 5-03~5-06 / 5-08~5-12 / 5-14~5-22 | ✅ 20 天 success | 正常签到 |
| 5-02 | ❌ HTTP 502 | server 自己挂 |
| 5-07 / 5-13 | ❓ 无记录 | cron 未触发 或 server 完全无响应 |
| **5-23** | ❌ sign 接口返**空 body** | **JM 反爬升级**:对自动化 sign 调用返 HTTP 200 + empty body(惩罚 + 迷惑) |

→ **核心症结**:即使 v2 cookie 完整、httpx 没问题、网络通畅,**server 端反爬**仍会对**网页端 `18comic.vip` 域名**的自动化 sign 调用静默返空。

### 2.3 三个旧方案的共同症结

| 版本 | 流程 | 失败点 |
|---|---|---|
| v1 selenium | Xvfb + Chrome UC 过 CF → httpx 调 sign | 同 IP 同天 2 次必拒 + sign 反爬同样存在 |
| v2 cookie 复用 | 抓 cookie + httpx 调 sign | **httpx TLS 指纹非 Chrome → server 反爬识别** |
| v3 账密+cf_clearance(未实施) | 每次重 login + httpx 调 sign | 同 v2,httpx TLS 仍非 Chrome |

→ **共性**:三个版本都死在**网页端 `18comic.vip/ajax/user_daily_sign`** 这一个接口上。

## 3. 关键新认知(本次调研最重要的发现)

### 3.1 **18comic 实际有两套域名,反爬强度天差地别**

**网页端 HTML 域名**(`DOMAIN_HTML_LIST`):
- `18comic.vip` / `18comic.org` / `jm-comic2.cc` / ...
- **受 CF Turnstile JS challenge 保护**(`cdn-cgi/challenge-platform/*`)
- **业务层反爬**(对非 Chrome TLS 指纹 + 异常签到节奏返空 body)
- 适合**人**访问,**对自动化极其敌对**

**API 移动端域名**(`DOMAIN_API_LIST`):
- `cdnhjk.net` / `cdngwc.cc` / `cdngwc.net` / `cdn-msp.jmapiproxy1.cc` / `cdn-msp.jmapiproxy2.cc` / `cdn-msp.jmapinodeudzn.net` / ...
- **几乎没有 CF Turnstile**(因为 mobile app 不能过 JS challenge)
- 业务层反爬弱(mobile app 需要正常访问)
- **专为 app 程序化调用设计**
- ✅ jmcomic 库 2.6.18 默认配置:`DEFAULT_CLIENT_IMPL = 'api'`(走 API 域名)

### 3.2 **`/ajax/user_daily_sign` endpoint 在两套域名都存在**

这是关键 — 同一个 endpoint,只是域名换了 → **反爬强度立刻从地狱模式变成简单模式**。

```python
# 网页端(被反爬)
POST https://18comic.vip/ajax/user_daily_sign
→ HTTP 200 + body="" (反爬静默拒绝)

# API 移动端(几乎不反爬)
POST https://www.cdnhjk.net/ajax/user_daily_sign
→ HTTP 200 + body='{"msg":"您已經完成每日簽到..."}'(正常)
```

(注:实际 endpoint 是 GET 还是 POST 需要 dry-run 确认。`pupusc/ComicsPuncher` 用的是 GET。)

### 3.4 jmcomic 库 2.6.18 的反爬基础设施(VPS-JM 已装)

`/usr/local/lib/python3.12/dist-packages/jmcomic/` 库源码关键点:

| 设施 | 位置 | 用途 |
|---|---|---|
| `DEFAULT_CLIENT_IMPL = 'api'` | `jm_config.py:DEFAULT_OPTION_DICT` | **默认走 API 端**,绕开网页 CF |
| `'type': 'curl_cffi'` + `'impersonate': 'chrome'` | `jm_config.py:DEFAULT_OPTION_DICT` | 默认用 curl_cffi 模拟 Chrome TLS 指纹 |
| `JmApiClient.login(user, pass)` | `jm_client_impl.py:794` | API 端登录,返回 JSON 含 `s` 字段(对应网页 `AVS` cookie) |

| `JmMagicConstants.APP_VERSION = '2.0.19'` | `jm_config.py:104` | 当前禁漫 App 版本号,自动随 server 升级 |
| `JmModuleConfig.DOMAIN_IMAGE_LIST` | `jm_config.py:140` | 多个 mobile API 域名 RR |

⚠️ **jmcomic 库本身没有 sign 实现** — 它主业是漫画下载。我们要做的就是:**借用它的 API 客户端基础设施 + 自己实现 sign endpoint 调用**(几十行代码)。

### 3.5 curl_cffi 对反爬的真实效果(2026-05 WebSearch 实证)

| 场景 | curl_cffi 是否有效 |
|---|---|
| TLS / JA3 / JA4 指纹检测 | ✅ 完全有效(模拟真实 Chrome) |
| HTTP header 顺序 / priority | ✅ 完全有效 |
| **CF Turnstile JS challenge** | ❌ 单独无效(需要 JS 执行 + 行为分析) |
| HTTP/2 帧顺序 | ✅ 有效 |

→ **curl_cffi 适用场景**:对**已经过 CF Turnstile 的 endpoint**(API 域名 / 已有 cf_clearance cookie)做调用,模拟真实 Chrome 避免业务层反爬识别。

→ **curl_cffi 不适用场景**:**首次过 CF Turnstile**(需要 selenium + UC 或商业方案)。

## 4. 5 个新方向调研

### 方向 1 · ⭐ 用 API 移动端域名替换网页域名(最低成本)

**修改**:`scripts/jmcomic/main.py` 把 `BASE_URL = "https://18comic.vip"` 改成 API 域名(`https://www.cdnhjk.net` 等),其余不动。

**工程量**:🟢 **极低**(2-4 小时)
- 改 `BASE_URL` + `INDEX_URL` + `SIGN_URL`
- 加 3-5 个 API 域名 fallback list,失败时按顺序换
- cookie 字段调整(API 域名可能不需要 `cf_clearance`)
- 改对应 README / manifest

**预期成功率**:**95-98%**(API 域名 CF 防护极弱,但偶尔 server 维护 / 域名失效)

**风险**:
- API 域名可能更换(jmcomic 库通过 `JM_REDIRECT_URL` 永久重定向 → 自动获取最新域名)
- cookie 可能要重新抓(API 域名的 cookie set 与网页端略不同)

**利弊**:简单、改动小,但失败时无兜底。

---

### 方向 2 · 🌟 三层回退策略(`pupusc/ComicsPuncher` 模式)

**模式**(完全照抄成熟开源项目):

```
Try 1: API 移动端域名 + 已有 cookie / 账密 login
  ↓ 失败
Try 2: HTML 网页端域名 + curl_cffi(impersonate=chrome124) + 已有 cookie
  ↓ 失败
Try 3: 直接 HTTP requests + 已有 cookie + 域名探测(返 200 的第一个)
  ↓ 失败
报失败 + 通知(Telegram / 微信)
```

每一层失败时自动降级到下一层。

**工程量**:🟢🟡 **中低**(6-10 小时)
- 抄改 `pupusc/ComicsPuncher/jm_punch.py` 适配我们的 manifest 协议(`run(config, context) -> RunResult`)
- 接入 jmcomic 库的 `JmApiClient.login()`(可选,跳过手动 cookie)
- 把 v2 已有的精细诊断(JmSignEndpointError 等 6 类异常)保留到第 3 层失败兜底
- 改 `requirements.txt` 加 `jmcomic>=2.6.18` 和 `curl_cffi>=0.7`
- 改 `manifest.yaml` 字段(改为账密 + 可选 cookie)
- 测试:本地 dry-run + scp 到 VPS-JM(等 IP 信任分恢复后)

**预期成功率**:**≥ 99%**(三层各自独立 fallback,理论上只有 server 完全挂才会全失败)

**风险**:
- 工程量比方向 1 大 3-5x
- 三层都失败时需要正确报错(已有 v2 的精细诊断可复用)

**利弊**:**银弹**,有完整开源项目验证,成熟稳定。

**工程量**:🟡 **中**(8-12 小时)
- 深入 jmcomic 库 API,理解 `req_api` / `decide_headers_and_ts` 签名机制
- 写自定义 sign 方法,可能需要 patch jmcomic 库或继承 JmApiClient
- 测试

**预期成功率**:**95-99%**(取决于自实现 sign 调用的正确性)

**风险**:
- jmcomic 库更新可能破坏自实现(主版本 2.6 → 3.x 风险)
- 库内部 API 不稳定(`req_api` 的 timestamp 签名机制 / `APP_TOKEN_SECRET` 等)

**利弊**:深度集成 jmcomic,享受其反爬基础设施全部优势;但实现比方向 2 更耦合。

### 方向 5 · 弹性机制叠加(任何方案都该加)

**不是独立方案,是任何方案的加固**:

#### 5.1 失败重试 + 24h 自动补签
```python
# manifest.yaml 加字段
retry_window_hours: 24
retry_interval_minutes: 30
```
- 首次失败 → 等 30min 重试一次
- 仍失败 → 等 1h 再试,渐进退避
- 24h 内无成功 → Telegram 告警

#### 5.4 Telegram / 微信告警
- v2 失败现在只在 /runs 详情页可见(用户不一定每天查)

## 7. 实施草图(方向 2)

### 7.1 新版 `scripts/jmcomic/main.py` 伪代码(总 ~500 行)

```python
"""JMComic 签到 v3 — 三层回退 + curl_cffi + jmcomic 库 API 端

设计哲学:
- 不再死磕网页端 18comic.vip(CF Turnstile + 业务反爬双重保护)
- 优先走 API 移动端域名(cdnhjk.net 等,CF 防护宽松)
- 三层独立 fallback,任一层成功立返
- 失败时继承 v2 精细诊断 + 首页 marker 兜底

借鉴:pupusc/ComicsPuncher/jm_punch.py(2026-05-09 commit)
"""

# 三层域名 fallback
FALLBACK_API_DOMAINS = [
    "www.cdnhjk.net",
    "www.cdngwc.cc",
    "www.cdngwc.net",
    "cdn-msp.jmapiproxy1.cc",  # jmcomic 库默认
    "cdn-msp.jmapiproxy2.cc",
]
FALLBACK_HTML_DOMAINS = [
    "18comic.vip",
    "18comic.org",
    "jm-comic2.cc",
]

# 三层签到函数 -- 来源 pupusc/ComicsPuncher
class JmSigner:
    def __init__(self, config, context):
        self.cookie_str = config.get("cookie", "")  # 可选,优先用
        self.username = config.get("username", "")  # 可选,API login 用
        self.password = config.get("password", "")  # 可选,API login 用
        self.logger = context.logger
        # ... 其它

    def run(self) -> RunResult:
        strategies = [
            ("API移动端", self._try_via_api_domain),
            ("HTML网页端 curl_cffi", self._try_via_html_curl_cffi),
            ("直接 HTTP cookie", self._try_via_direct_http),
        ]
        last_diag = None
        for name, fn in strategies:
            try:
                result = fn()
                if result.success:
                    return result
                last_diag = result.data
            except Exception as e:
                self.logger.warning(f"策略 [{name}] 失败: {e}")
                last_diag = {"strategy": name, "error": str(e)}

        # 三层都失败 -- 兜底首页 marker(v2 逻辑)
        already_signed = self._check_via_index_marker()
        if already_signed:
            return RunResult(success=True, message="今日已签到(首页 marker 兜底)", data=...)

        return RunResult(success=False, message="三层策略均失败", data=last_diag)

    # 策略 1:API 域名 + 已有 cookie(优先)/账密 login(次选)
    def _try_via_api_domain(self) -> RunResult:
        from curl_cffi import requests as cc_req
        session = cc_req.Session(impersonate="chrome124")

        if self.cookie_str:
            # 注入已有 cookie
            for k, v in parse_cookie_string(self.cookie_str).items():
                session.cookies.set(k, v)
        elif self.username and self.password:
            # 用 jmcomic 库走 API 端 login(完全绕开 CF)
            self._api_login_with_jmcomic(session)
        else:
            raise JmCookieFieldError("cookie 和 账密 至少要有一个")

        for domain in FALLBACK_API_DOMAINS:
            try:
                resp = session.get(
                    f"https://{domain}/ajax/user_daily_sign",
                    timeout=20,
                    headers={
```

```python
import json
import logging
import random
import time

from jmcomic import JmOption


# ---- 网络容错配置 ----
REQUEST_TIMEOUT = 25
MAX_RETRIES = 2
RETRY_BACKOFF_BASE = 2

FALLBACK_HTML_DOMAINS = ["18comic.vip", "18comic.org"]
FALLBACK_API_DOMAINS = ["www.cdnhjk.net", "www.cdngwc.cc", "www.cdngwc.net"]

_NON_RETRYABLE_KW = ("403", "401", "Forbidden", "Unauthorized")


def _is_retryable(error_msg):
    return not any(kw in str(error_msg) for kw in _NON_RETRYABLE_KW)


def _retry_with_backoff(func, max_retries=MAX_RETRIES):
    last_exc = None
    for attempt in range(max_retries + 1):
        try:
            return func()
        except Exception as e:
            last_exc = e
            if attempt == max_retries or not _is_retryable(str(e)):
                raise
            delay = RETRY_BACKOFF_BASE * (2 ** attempt) + random.uniform(0, 1)
            logging.warning(f"请求失败 (第 {attempt + 1}/{max_retries + 1} 次): {e}，{delay:.1f}s 后重试")
            time.sleep(delay)
    raise last_exc


class JmPuncher:
    """三层回退策略:HTML / API / 直接 HTTP"""

    def __init__(self, username, password, proxy=None):
        self.username = username
        self.password = password
        self.proxy = proxy

    def run(self):
        strategies = [
            ("HTML网页端", self._try_via_html_client),
            ("API移动端", self._try_via_api_client),
            ("直接HTTP(兜底)", self._try_via_direct_http),
        ]
        for name, strategy in strategies:
            try:
                logging.info(f"尝试策略: {name}")
                strategy()
                return
            except Exception as e:
                logging.warning(f"策略 [{name}] 失败: {e}")
        logging.error("所有签到策略均失败")
        print("JM 签到失败: 所有策略均无法完成签到")

    def _try_via_api_client(self):
        # API 域名通常 Cloudflare 防护较宽松，GitHub Actions 可连通
        client = self._build_client("api", FALLBACK_API_DOMAINS)
        self._do_login(client)

        domain = client.get_domain_list()[0]
        logging.info(f"使用 API 域名 [{domain}] 执行签到")

        resp = _retry_with_backoff(
            lambda: client.get(
                "/ajax/user_daily_sign",
                timeout=REQUEST_TIMEOUT,
                headers={
                    "Referer": f"https://{domain}/",
                    "X-Requested-With": "XMLHttpRequest",
                    "Accept": "application/json, text/plain, */*",
                },
            )
        )
        self._parse_and_handle(resp.text)

    def _build_client(self, impl, fallback_domains):
        postman_type = self._detect_postman_type()
        meta_data = {"timeout": REQUEST_TIMEOUT}
        if postman_type == "curl_cffi":
            meta_data["impersonate"] = "chrome124"
        if self.proxy:
            meta_data["proxies"] = {"http": self.proxy, "https": self.proxy}

        make_config = lambda domains: {
            "client": {
                "impl": impl,
                "domain": domains,
                "retry_times": 1,
                "postman": {"type": postman_type, "meta_data": meta_data},
            }
        }
        try:
            option = JmOption.construct(make_config([]))
            return option.build_jm_client()
        except Exception:
            logging.warning(f"动态域名解析失败，使用预置域名: {fallback_domains}")
            option = JmOption.construct(make_config(fallback_domains))
            return option.build_jm_client()

    @staticmethod
    def _detect_postman_type():
        try:
            import curl_cffi  # noqa: F401
            return "curl_cffi"
        except ImportError:
            return "requests"

    def _do_login(self, client):
        logging.info(f"正在登录禁漫 (用户: {self.username})...")
        def action():
            resp = client.login(self.username, self.password)
            cookies = _resp_cookies(resp)
            if not any(k in cookies for k in ("remember_id", "remember", "yuo1", "AVS")):
                raise RuntimeError("登录响应未包含有效会话 Cookie")
            return resp
        _retry_with_backoff(action)
        logging.info(f"登录成功 (用户: {self.username})")

    def _handle_sign_result(self, data):
        if data.get("error") == "finished":
            print("今日已签到")
        elif data.get("errorMsg") == "Not legal.ajax":
            print("登录验证未通过")
        elif "msg" in data:
            print(f"签到结果: {data['msg']}")
        else:
            print(f"签到状态: {data}")
```
