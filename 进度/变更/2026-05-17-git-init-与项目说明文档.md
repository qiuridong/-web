## 完整动作链

### 1. 安全加固 `.gitignore`

**问题**:原规则只有 `/data/`(项目规划路径),但 dev/test 实际产生的数据在 `backend/data/`,**该目录从未被 ignore**。直接 `git add .` 会把:
- `backend/data/encryption.key`(Fernet 主密钥,44 字节)
- `backend/data/db.sqlite3`(含用户 bcrypt hash + 实例加密 cookie,144KB)

加进 stage,push 上去 = 主密钥**永久泄露**到 GitHub(即使 force push 删,git history 还在)。

**修复**(`.gitignore` 第 8-19 行附近):

```diff
 # ===== 应用运行时数据 =====
+# ⚠️ 关键!encryption.key 是主密钥(Fernet),db.sqlite3 含用户密码 hash + 加密 cookie
+# 两个路径都要拦:根 data/ 是规划路径,backend/data/ 是 dev/test 实际产生路径
 /data/
 !/data/.gitkeep
+/backend/data/
+!/backend/data/.gitkeep

 # ===== 应用日志 =====
 /logs/
 !/logs/.gitkeep
+/backend/logs/
+!/backend/logs/.gitkeep
```

新建 `backend/data/.gitkeep`(空文件),让目录结构入 git。

**验证**:`git check-ignore -v` 跑了 6 个敏感路径,全部命中 .gitignore 规则;同时 7 个正常源码路径(`app/main.py` / `App.tsx` / `coklw/main.py` / README.md / 项目说明.md / 进度/README.md)无误伤。
