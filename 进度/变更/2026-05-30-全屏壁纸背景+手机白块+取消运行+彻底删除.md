## A · 壁纸铺满整个布局(侧栏 + 顶栏 + 内容)

### 问题
PR #8 把背景图只贴在内容滚动区 `<main>` 的内层 `min-h-full` 包装上 → 侧栏(`bg-card`)、
顶栏(`bg-background/70`)是不透明的纯色,壁纸只在中间内容区出现,视觉割裂。

### 方案(重构布局背景层)
- **背景图移到根容器**(`h-screen w-full overflow-hidden` 的那个 div):整屏铺满,
  因根容器 `overflow-hidden` + viewport 高,背景**等效 fixed**,内容在 `<main>` 内独立滚动。
- **遮罩层**(opacity 调亮暗 + blur 罩)提到根容器的第一个 absolute 子元素,`z-0`。
- **侧栏 + 顶栏 + 主区**都加 `relative z-10` 浮在遮罩之上。
- **侧栏 / 顶栏在有壁纸时半透明 + 毛玻璃**(`bg-card/65 backdrop-blur-xl` / `bg-background/55
  backdrop-blur-xl`)→ 壁纸透出来,整屏统一玻璃质感;无壁纸时回退纯色(`bg-card` / `bg-background/70`)。
- 通过 `<Sidebar translucent={hasBackground}>` 新增 prop 控制。

### 收益(顺带解决旧 bug)
- 不再需要 PR #8 的 `min-h-full` + 内层 overlay 那套 scroll 对齐 hack(背景现在 viewport 固定,
  本来就不随内容滚)。
- 滚动到任何位置都不会露出 raw 背景或白底。

## B · 手机端右侧白块

### 问题
用户报「手机端最右边是大部分白块」+「手机端也是只有内容有壁纸」。

### 根因 & 修复
白块本质是**非壁纸区**(根容器 `bg-background` = 浅色白)露出来:壁纸被困在内容区,
手机上内容更窄 / 偶有横向溢出 → 右侧露白。

A 的改动**直接修掉**:壁纸铺根容器 viewport 固定后,侧栏/顶栏/任何空隙/横向溢出露出的
都是**壁纸**而非白底。额外加固:`<main>` 由 `overflow-auto` 改 `overflow-y-auto
overflow-x-hidden` —— 防个别宽元素把整页撑出横向滚动(表格仍由 DataTable 自身 `overflow-auto` 内部横滚)。

## C · 取消运行中 / 排队中的任务

### 关键发现:后端 + hook 早就有,只缺 UI
- 后端 `POST /api/v1/runs/{id}/cancel`(`runs.py:364`)+ `run_service.cancel_run`(校验 pending/running
  → 翻 DB 状态 `cancelled` → 调 `scheduler.cancel_run` 杀本地子进程)**已存在**。
- 前端 hook `useCancelRun()`(`runs.ts:197`)**也已存在**,只是没有任何组件渲染按钮。
- → 用户「没有取消功能」= **UI 没接**,不是没实现。本次只补 UI。

### 实施:`<CancelRunButton>`(新组件)+ 接入 2 处
- 新建 `components/common/CancelRunButton.tsx`:仅当 status ∈ {pending, running} 渲染;
  按钮 + AlertDialog 二次确认 + `useCancelRun`。
- 接入 **实时日志 tab**(`ScriptLogConsole`)—— 用户看运行中 run 的主界面(就是这次的场景)。
- 接入 **运行详情抽屉**(`RunDetailSheet`)—— `/runs` 列表 + 实例执行历史点行进来都覆盖。

### 取消语义(弹窗里如实告知,避免误解)
| 场景 | 行为 |
|---|---|
| `pending`(还没派给节点) | 干净取消,永不执行 |
| `running` · **本地**节点 | executor 直接 SIGTERM→SIGKILL 子进程(真终止) |
| `running` · **远程** agent | **软取消**:仅翻 DB `cancelled`;agent 单线程串行,当前 run 子进程会跑完(上报时被 `already_cancelled` 识别后丢弃结果) |

### ⚠️ 已知限制(本次未做,见后续)
远程 agent(`signin_agent.py`)是**单线程串行**:`_execute()` 会阻塞 poll 循环直到子进程结束,
所以正在「随机延迟 sleep」中的远程 run **无法被中途真杀**。要做到远程真终止需把 `_execute`
改成后台线程 + 控制 poll 通道 + `cancel` pending_action —— **触及正在稳定签到的 jmcomic agent,
风险高,本次刻意不动**(用户多次强调别动 working jmcomic)。当前软取消对幂等的 jmcomic 够用。

## 7 · 关键经验

- **「没有这个功能」常常 = UI 没接**:取消运行后端 + hook 全有,只是没按钮;删除也是 hook 有没 wire。
  排查用户「缺功能」诉求,先 grep 后端 endpoint + 前端 hook,可能只差最后一公里。
- **壁纸放最外层 viewport 固定**是「整屏统一背景 + 任何空隙不露白」最省事的架构 —— 比给每个面板
  单独贴背景干净,顺带把手机白块一起修了。
- **单线程 agent 限制了远程真取消** —— 记录为已知限制,不为了「完整」去动正在稳定工作的 jmcomic agent。
