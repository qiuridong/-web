## 关键解读

### 灰云直连意味着什么(对部署很关键)
- Caddy 的自动 HTTPS 走 Let's Encrypt **HTTP-01 challenge**(请求 `http://jb.aijiaxia.cc/.well-known/acme-challenge/...`)— 必须能直达源站,Cloudflare 橙云会代理掉 HTTP 请求,导致 challenge 失败
- 现在灰云模式 = DNS 直接返回 `154.9.238.144`,Caddy 拿到 challenge 请求 → 自动响应 → 拿证书 → 完成 HTTPS
- 部署后访问 `https://jb.aijiaxia.cc` 即可,**无须任何手动证书操作**

### 密钥使用
- OpenSSH RSA 格式
- 本地连接:`ssh -i "J:\密钥\美国质量8-8\vcs-deploy-rsa" root@154.9.238.144`
- Windows 下密钥若权限太开放,ssh 会拒绝使用;首次连接前可能需要 `icacls` 调整权限(或用 Git Bash 跑)
- ⚠️ **密钥文件不入 git**(它在 `J:\密钥\` 外部目录,天然安全),只在进度文档里记**路径**作为接手参考
