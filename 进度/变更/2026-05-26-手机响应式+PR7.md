## 1. 实施

### 1.1 `components/layout/AppLayout.tsx`(+55 / -7)— mobile sidebar 抽屉

**前**:CSS Grid 2 列(sidebar 64-240px + main 1fr),任何 viewport sidebar 都占位,手机 320-414px 上 main 被严重挤压(只剩 ~256px)。

**后**:
- mobile (< md):`flex`(单列布局)+ sidebar 改 Sheet 抽屉(从左滑入,backdrop 点击关)
- desktop (≥ md):`grid` + 现有 CSS Grid 2 列保持
- 加 `mobileSidebarOpen` state + Topbar 汉堡按钮(`<Menu>` icon,`md:hidden`)触发抽屉
- 路由切换时 `useEffect on location.pathname` 自动关 mobile sidebar(用户点导航不需手动关)
- mobile sidebar 始终展开态(不折叠,`collapsed={false}`),点 logo 关抽屉(`onToggle` 重载为 close)

```tsx
{/* Desktop sidebar — md+ 直接渲染在 grid 第 1 列 */}
<div className="hidden md:block">
  <Sidebar collapsed={sidebarCollapsed} onToggle={...} />
</div>

{/* Mobile sidebar — < md 用 Sheet 抽屉 */}
<Sheet open={mobileSidebarOpen} onOpenChange={setMobileSidebarOpen}>
  <SheetContent side="left" className="w-[260px] border-r-0 p-0 md:hidden">
    <Sidebar collapsed={false} onToggle={() => setMobileSidebarOpen(false)} />
  </SheetContent>
</Sheet>
```

**Topbar 也改**:
- 加汉堡按钮(`md:hidden`)在 mobile 左侧
- 搜索框 mobile 上文案简化:`<span className="hidden sm:inline">搜索 / 跳转 / 命令…</span>` + `<span className="sm:hidden">搜索</span>`
- Topbar padding `px-3 sm:px-4 lg:px-6` mobile-first
- 主区滚动容器 `px-3 py-4 sm:px-6 sm:py-6 lg:px-8`(原 `px-4 py-6`)

### 1.2 `components/ui/sheet.tsx`(+2 / -2)— 全局 mobile 全宽

**前**:`left/right` 侧 Sheet 默认 `w-3/4 sm:max-w-sm` — mobile 75% 宽,320px 屏只剩 240px,内容挤;sm+ max-w 384px。

**后**:`w-[calc(100vw-1rem)] sm:w-3/4 sm:max-w-sm` — mobile 接近全宽(留 1rem 边距 = 16px),sm+ 保持 75% + max-w-sm。

影响范围:所有 `<Sheet side="left|right">` 用户(InstanceFormSheet / RunDetailSheet / NotificationHub / ScriptDevGuideSheet / ScriptList / 新加的 mobile sidebar)— 全局生效,**这一行修复 7+ 个组件的 mobile 体验**。

### 1.3 `components/common/DataTable.tsx`(+3 / -1)— mobile 横滚

**前**:`<div className="overflow-hidden rounded-xl ...">` 包 `<Table>` — mobile 列多(脚本列表 7 列 / 节点列表 6 列等)时,列被 hidden 切掉,完全看不见。

**后**:嵌套外层 `overflow-hidden`(保 border-radius)+ 内层 `overflow-x-auto`(允许横滚)。Table 自动有 min-width(根据列内容),mobile 上手指横滑就能看到全部列。

```tsx
<div className="overflow-hidden rounded-xl border border-border bg-card shadow-xs">
  <div className="overflow-x-auto">                {/* 新加 */}
    <Table>...</Table>
  </div>
</div>
```

## 5. 设计选择

| 项 | 选择 | 理由 |
|---|---|---|
| Mobile 断点 | `md=768px`(Tailwind 默认) | iPad portrait (768px) 之下都算 mobile |
| Sidebar mobile | Sheet 抽屉(从左) | Material/iOS 通用模式,backdrop 关 |
| Sidebar mobile 折叠态 | 不支持(始终展开) | mobile 空间换功能,抽屉本来就是临时显示 |
| 路由切换关抽屉 | useEffect on pathname | 用户点导航不需手动关 |
| Sheet mobile 宽 | `calc(100vw-1rem)` | 留 1rem 视觉边距,避免贴边 |
| DataTable mobile | 横滚(`overflow-x-auto`) | 比"卡片视图切换"轻量,实现成本 0,数据完整可见 |
