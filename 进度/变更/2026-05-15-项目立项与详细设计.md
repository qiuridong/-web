# 2026-05-15 · 项目立项与详细设计

## 起因

用户提出需要一个"签到脚本聚合 web 界面",能管理自己制作的多个签到脚本,部署在服务器上后通过浏览器统一管理。

## 需求确认

通过两轮对话明确以下要求:

| 需求点 | 用户决定 |
|---|---|
| 部署目标 | Ubuntu 24 系统 + Docker |
| 脚本生态 | **全自建**(不对接青龙面板) |
| UI 要求 | **必须美化**(不接受默认 Element Plus 灰扁平) |
| 工作方式 | **多 agent 并行协作**,Claude 负责统筹 |
| 脚本语言(默认推断) | Python(全自建语境下,统一 Python 集成最丝滑) |
| 用户规模(默认推断) | 单用户 / 小团队,bcrypt + session 即可,不做 RBAC |

## 技术栈最终选型

- 后端:Python 3.11 + FastAPI + APScheduler 3.x + SQLAlchemy 2.x + SQLite (WAL) + apprise + cryptography.Fernet + bcrypt
- 前端:Vue 3 + Vite + TypeScript + Pinia + Vue Router 4 + Element Plus(深度美化)+ openapi-typescript
- 部署:Docker Compose + Caddy 2(自动 HTTPS)+ restic 备份

详见 [决策.md](../决策.md) 全部 10 条 ADR。

### 后端架构覆盖
- 8 张表完整 schema(`scripts` / `instances` / `runs` / `users` / `sessions` / `notification_channels` / `notification_rules` / `settings`)
- 36 个 REST API 端点(全部带鉴权标识、请求/响应字段)
- manifest.yaml 完整 schema + 11 种字段类型系统
- main.py 契约(`run(config, context) -> RunResult`)
- 调度引擎执行流程(14 步)
- Fernet key 管理 + 字段加密 + secret 自动脱敏规则
- 后端目录树 + 32 个模块的职责说明
- pyproject.toml 依赖清单
- 9 类异常体系 + 标准错误响应格式

### 前端 UI 设计覆盖
- 5 词调性(克制/现代/精致/安静/可信赖)
- 完整深浅色配色(brand Indigo + accent Teal + 4 语义色 + 中性色阶)
- 8 档字号 + 间距 token + 圆角 + 阴影 + 动画 token
- 14 个路由 + 鉴权 guard 流程
- 11 个关键页面 wireframe(含登录/setup/仪表盘/脚本列表/脚本详情/实例配置/历史/SSE 日志查看器/通知/设置/全局)
- 10 个公共组件清单
- Pinia stores 划分
- 美化关键手法(玻璃拟态、呼吸光晕、tabular-nums、骨架屏、⌘K 命令面板)
- 推荐目录结构 + 关键依赖

### 技术调研覆盖
- APScheduler 3.x vs 4.x 选型(明确 4.x 不可用)
- AsyncIOScheduler + FastAPI lifespan 集成代码骨架
- SQLite WAL + check_same_thread + busy_timeout PRAGMA 配置
- `asyncio.create_subprocess_exec` + `start_new_session=True` + 进程组超时强杀完整代码
- apprise 1.10+ 国内通知渠道(Server酱/钉钉/飞书/企微/Bark)URL 模板
- create-vue + Element Plus + 按需引入 + dark mode + theme 覆盖完整 vite.config.ts
- sse-starlette 子进程 stdout → SSE 桥接代码
- Caddy 2 vs Nginx 比较与完整 Caddyfile + docker-compose.yml + Dockerfile 骨架
- restic 备份脚本
- Fernet + MultiFernet 密钥轮换 + SQLAlchemy TypeDecorator 字段加密
