### 1.3 改造 `UploadScriptDialog.tsx`(+280 行)

**新增 helper `analyzeUpload(files: File[]): Promise<FileAnalysis>`**:
- 来源分两种:`files.length === 1 && .zip` → `JSZip.loadAsync` 解压;否则 → 遍历 `File.webkitRelativePath`
- 对每个 `SCRIPT_REQUIRED_FILES`,以 basename 在**最浅深度**位置查找(避免误判 `subfolder/manifest.yaml`)
- 解析最浅 manifest.yaml 内容 → `yaml.load` → 提取 slug / name / version / fields.length
- 返回 `{ checklist, manifestSummary, yamlError, extraPyCount }`

**新增 state**:`analysis` / `analyzing` / `analyzeError` / `guideOpen` / `downloadingTemplate`,`useEffect` 监听 `files` 变化触发 `analyzeUpload`(带 cancelled flag 防 race)

**新增 UI**:
- 顶部 actions bar(在 idle 分支最顶):带说明文字 "第一次写脚本?先下载模板项目,改改 manifest + main.py 即可" + 两个 outline 按钮(下载模板 / 开发指南)
- 拖拽区之后 / 文件预览之前:新组件 `FileChecklist`,显示分析状态(loading / error / analysis)
  - 5 项清单(每项 ✅/❌/⚪ 图标 + 必填/可选标签 + hint 说明)
  - manifest 摘要绿框(slug / name / version / fields 数)或 yaml 错误红框
  - 缺必填红框列出具体缺哪些
- Dialog 包成 `<>...<ScriptDevGuideSheet open={guideOpen} ... /></>` 同级 portal

**修改 `canSubmit`**:加上 `!analyzing && missingRequired.length === 0 && !analyzeError`

**修复 TS strict**:`files[0]` 在 strict noUncheckedIndexedAccess 下需要中间变量(`const single = files.length === 1 ? files[0] : undefined`)

## 6. 教训与设计要点

- **客户端预解析远胜后端校验**:之前开发者拖错 zip 要等后端 422 才知道缺什么;现在拖完立刻看清单 ✅/❌。开发者上传次数预期从 3-5 次降到 1 次。
- **jszip 双用**:既用于解压(分析上传 zip)又用于压缩(动态生成模板 zip),只装一份依赖,~100KB。
- **模板项目放前端 vs 后端**:本来想加 `GET /scripts/template.zip` endpoint,实测前端 jszip 动态生成更简单(不需要后端来回 + Cors / Auth 复杂度 + 修模板得 docker rebuild),适合纯静态资源。
- **Sheet 模式复用**:5-25 凌晨给 NotificationHub 设计的 `flex-col gap-0 p-0` + `min-h-0 flex-1 overflow-y-auto` + `scrollbar-gutter:stable` + sticky footer 模式被 ScriptDevGuideSheet 直接复用,长内容也能滚 + 关闭按钮始终可见。
- **PR review 流程开启**:之前 main 直接 push 是单人开发期的速度优先,现在引入 PR 是为接下来"别的开发者上传脚本"的协作场景做准备 — 自己提的功能,自己最先享受协作流程。
