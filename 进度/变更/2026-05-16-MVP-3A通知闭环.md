## 实现细节

### A. dashboard_service.get_overview 加 4 字段

`backend/app/services/dashboard_service.py`(+125 行):

- **`sparkline_7d_success: list[float]`** 长度 7,过去 7 天每日成功率(0.0-1.0),末位是今天
- **`sparkline_7d_runs: list[int]`** 长度 7,过去 7 天每日 run 总数
- **`notifications_24h: int`** 过去 24h `count(notification_rules) WHERE last_fired_at > now - 24h`
- **`next_run_at: datetime | None`** `SELECT MIN(next_run_at) FROM instances WHERE enabled=true AND next_run_at IS NOT NULL`

实现方式:
- sparkline 用 SQLite `strftime('%Y-%m-%d', started_at)` 按本地日分桶,Python 端补齐 7 天空桶。成功率分母**只看终态**(success/failure/error/timeout/cancelled),与 24h/7d 现有口径一致
- notifications_24h 用 try/except 兜底,字段缺失/SQL 异常时降级返 0
- next_run_at:SQLAlchemy `func.min()`;若返裸 datetime 无 tz 归一到 UTC

### B. schemas/dashboard.py 同步

`backend/app/schemas/dashboard.py`:`DashboardOverview` 加 4 字段(都 Field validator 严约束)。

```python
sparkline_7d_success: list[float] = Field(..., min_length=7, max_length=7, ...)
sparkline_7d_runs: list[int]   = Field(..., min_length=7, max_length=7, ...)
notifications_24h: int          = Field(..., ge=0, ...)
next_run_at: datetime | None    = Field(None, ...)
```

### C. executor → dispatcher 接通

`backend/app/scheduler/executor.py`:

1. **删掉**旧的占位调用(用错名的 `dispatch_run_result` + `get_event_loop().run_in_executor`)
2. **新增** `_dispatch_notification(run_id, event)`:
   - 自己拿 SessionLocal session(避免污染外层 session)
   - import `dispatch_run_event` + `get_cipher` + `get_pool` 单例
   - 把 `event` 直接传 `run.status`(已是 success/failure/error/timeout 之一,cancelled 在 dispatcher 内被静默)
   - 整体 try/except,任何异常只 warning,不影响 executor 主流程
3. **关键决策**:把 retry 判断**前移**到 dispatch 前。`will_retry=True` 时不发通知。这样 retry 链中途的失败不会刷屏,只在最后一次 attempt 或本来就不重试时发(对应 § 4.5 "最终通知按最后一次结果触发")
