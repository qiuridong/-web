3. **补 `_chunked_sleep(total_sec, chunk=30)` 函数**(放在 `_safe_logout` 之后,line ~525):
   ```python
   def _chunked_sleep(total_sec: int, chunk: int = 30) -> None:
       """分块 sleep,降低长延迟期间无法响应中断的风险。
       主程序 SIGTERM 时,默认 time.sleep(N) 整段无法响应,要等满 N 秒。
       改用 30 秒 chunk 后,SIGTERM 最多 30 秒内被识别。
       """
       remaining = total_sec
       while remaining > 0:
           time.sleep(min(chunk, remaining))
           remaining -= chunk
   ```
4. **替换 run() 内 4 处 time.sleep 用 _chunked_sleep**:
   - line ~545:随机延迟主 sleep
   - line ~610 / ~620 / ~630 / ~640:4 处重试间隔 sleep

## 经验教训

| 教训 | 适用场景 |
|---|---|
| 部署位置不要假设,**用户的"主面板管理"≠"主面板节点跑"**(用户可能想"主面板 web UI" + "其它节点 worker") | 任何涉及多节点 / 远程的需求,先问"部署到哪台机器,哪台 web 显示" |
| **CF 信任分** 不能被探活 / sandbox 测试浪费 — 每次测试都是 1 次消耗 | 反爬类脚本的 dev/test 要在 mock/fake server 上,不在生产 server 上 |
| **uid/gid 对齐很重要**:scp 默认 root owner 但容器用户可能 1000 | 任何 Docker 容器化项目,scp 后立刻 chown |
| **日志要看时间戳 + 完整上下文**,别只看末尾几行 | 调试 scanner / scheduler 这类周期任务 |
| Agent 派出后,**主对话 thread 不能动 agent 在改的文件**(backend/ / frontend/ / 进度/README.md / 进度/变更/MVP-1.md) | 任何 background agent 并行场景 |
