### 1.2 `NotificationHub.tsx` 改造

### 1.3 Sheet 滚动修复(2 个 Sheet 共改)

**问题** · 字段过多时(尤其 RuleSheet 加了字段速查 Card)内容溢出 SheetContent,**保存/取消按钮被遮挡**。

**改造** · 把两个 Sheet 改成 header/scroll/footer 三段式:

```
<SheetContent flex flex-col gap-0 p-0 sm:max-w-md>
  <SheetHeader shrink-0 border-b px-6 pb-4 pt-6>...</SheetHeader>
  <form flex min-h-0 flex-1 flex-col>
    <div flex-1 space-y-5 overflow-y-auto px-6 py-5 [scrollbar-gutter:stable]>
      ...所有字段...
    </div>
    <SheetFooter shrink-0 border-t bg-background px-6 py-4 flex-row-reverse>
      ...按钮永远钉底...
    </SheetFooter>
  </form>
</SheetContent>
```

**关键 Tailwind**:
- `min-h-0` 让 flex 容器子元素能正常溢出滚动(必备 trick,否则 flex-1 子元素不会真滚)
- `shrink-0` 在 header/footer 上,防被滚动内容挤压
- `scrollbar-gutter:stable` 让滚动条空间始终预留(即使内容不溢出),视觉上明确"这里可滚"
- `bg-background` + `border-t` 让 footer 看起来像独立 sticky 栏

### 1.4 Select 下拉 max-height 修复

**问题** · 用户截图显示:渠道类型预设(13 项)展开后,下拉向下完全展开,**遮挡了 Apprise URL/备注/启用/保存按钮**。

**根因** · shadcn `<SelectContent>` 默认只有 `max-h-[--radix-select-content-available-height]`(可用空间),不限制内容长度。

**修复** · `NotificationHub.tsx` 内全部 7 个 `<SelectContent>` 用 `replace_all` 一次加 `className="max-h-[280px]"`,让下拉最多 280px 高然后内部上下箭头滚动。

## 2. 脚本同步 Pull 方案 — 设计(未实施)

### 2.1 用户问题

> "假如现在我有一个 zip python 脚本,那么我部署的指定 vps 上去,似乎没有这样的功能,帮我想想这个是否要加上,如何实现?"

### 2.2 现状摸底(grep + 读 4 文件)

| 层 | 现状 | 缺什么 |
|---|---|---|
| 主面板侧 | ✅ MVP-5 已有 `POST /scripts/upload`(zip / multipart),已有在线编辑、dry-run、scan_all 入库 | — |
| **主面板 → agent 同步** | ❌ **完全没有**。`backend/app/api/v1/agent.py` 只有 4 个端点:poll/stdout/result/heartbeat,**没有 scripts 分发端点** | bundle 下发 API |
| Agent 侧 | ❌ scripts 靠手动 `scp -r /opt/signin-panel/scripts/jmcomic root@VPS:/opt/signin-agent/scripts/`(README/故障排查都明写) | 自动拉脚本逻辑 |

→ 用户上传 zip 到主面板 → 主面板自己 scripts/ 有了 → **agent 还是空的** → 跑 task 时 agent 找不到 main.py 报错。

### 2.3 推荐方案 · Pull 模型

复用现有 MVP-1 长轮询架构,加 2 个 agent 端点 + agent 端 ~80 行代码:

```
backend agent.py + 2 endpoint
├─ GET /agent/scripts/<slug>/manifest    返 {version, files[], bundle_sha256, bundle_size}
└─ GET /agent/scripts/<slug>/bundle.zip  返 zip 二进制(过滤 .backups/__pycache__/data/.git)

agent signin_agent.py +1 函数
└─ _ensure_script_synced(slug):
    1. 比对本地 .version_marker vs 远端 manifest.bundle_sha256
    2. 不一致 → 拉 bundle → 校验 sha256 → 原子解压(.tmp → rename)
    3. 写新 .version_marker
   在 _execute_task() 开头调,sandbox_runner 之前
```

**触发链路天然自动**:用户 web 改 manifest 保存 → scan_all 入库新 version → agent 下次 poll 拿到的 task payload 里 `script_version` 变了 → 触发 hash 比对 → 拉新 bundle。

**安全已 cover**:bundle 端点走现有 `AgentNode` Bearer Token middleware;ZipSlip 防护、大小上限、文件过滤(复用 `script_upload_service` 的常量)。

### 2.4 工作量评估

| 模块 | 改动量 | 风险 |

| backend agent.py | +60 行(2 个端点 + bundle 缓存) | 低,只加端点 |
| backend manifest 计算 | +20 行(hashlib + zipfile) | 低 |

1. 本会话已部署 4 轮前端改动,继续动 agent 会模糊"明早 cron 失败的根因"

## 6. 关键约束/教训

- **Auto Mode 下也要止步问 SSH 凭证** — 不能盲试多个 ssh key(被 auto mode 拦,合理)
- **shadcn Sheet 默认无 sticky footer** — `SheetContent` 默认 `p-6` 单层,需手动拆 header/scroll/footer 三段式 + `min-h-0` trick
- **Radix SelectContent 不限制 max-h** — 仅给 `--radix-select-content-available-height`,长列表会展满 viewport;**长选项列表必须显式 `max-h-[280px]` + overflow-y-auto**
- **scrollbar-gutter:stable 是认知 UX 加固** — 即使内容不溢出也预留滚动条空间,用户视觉确认"这区域支持滚动",减少"是不是没改对"的疑虑
