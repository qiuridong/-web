## 部署步骤(实际执行)

```bash
# 本地
tar czf - --exclude='...' . | ssh root@154.9.238.144 'tar xzf - -C /opt/signin-panel'

# 服务器
cd /opt/signin-panel
apt install -y docker.io docker-compose-v2 python3-certbot-nginx
cat > .env <<EOF   # UID=0 / GID=0 / DOMAIN=jb.aijiaxia.cc / ACME_EMAIL=...
cp deploy/nginx-jb.aijiaxia.cc.conf /etc/nginx/sites-available/jb.aijiaxia.cc
ln -sf /etc/nginx/sites-available/jb.aijiaxia.cc /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
certbot --nginx -d jb.aijiaxia.cc --redirect --agree-tos -m yunkelai12251@gmail.com -n
docker compose build && docker compose up -d
sleep 8 && docker compose exec -T backend python -m alembic upgrade head
```

## Dockerfile build 修了 2 个 bug

1. **bash 进程替换**:`RUN ... -r <(uv export ...)` 在 `/bin/sh` 不支持,改两步:`uv export > /tmp/req.txt && uv pip install -r /tmp/req.txt`
2. **COPY README.md 缺失**:`pyproject.toml` 的 `readme = "README.md"` 字段触发 hatchling 解析时要求文件存在;builder 阶段只 COPY pyproject 没 COPY README,补 `COPY README.md ./` 即可
