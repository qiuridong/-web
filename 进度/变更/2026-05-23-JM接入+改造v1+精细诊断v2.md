### v2 加的能力

#### A. 6 个细分异常类(每个带 diag 字段)
```python
class JmError(Exception):
    def __init__(self, message, **diag): super().__init__(message); self.diag = diag
    def to_data(self): return {"error_class": type(self).__name__, **self.diag}

JmCloudflareBlocked / JmLoginEndpointError / JmLoginFailed
JmSignEndpointError / JmIndexEndpointError / JmNetworkError / JmAlreadySignedToday
```

#### B. `_http_request` helper:每次 HTTP 调用统一日志 + 结构化 diag
```python
def _http_request(session, method, url, label, *, logger, **kwargs):
    # 抓 status / content_type / content_length / elapsed_ms / body_preview
    # 网络错抛 JmNetworkError(exc_type/elapsed_ms)
    # 返 (response, diag dict)
    logger.info(f"[{label}] HTTP {status} ct={ct} len={length}B body_len={body_len} {elapsed}ms")
```

#### C. `_parse_json_or_raise`:精细 JSON 校验(替代笼统 `json.loads`)
```python
# 4 种具体原因分别 raise:
#   非 200 / 空 body / 非 JSON Content-Type / 解析失败
# 每种 raise 都带完整 diag → /runs 详情页直接看
```

#### D. **关键修复**:`_check_already_signed_via_index` 兜底
```python
# sign 接口空响应 / 非 JSON / 解析失败时,启动兜底:
#   GET https://18comic.vip/(首页)
#   扫 8 个繁简体 marker:
#     今日已簽到 / 已簽到 / 今日已签到 / 已签到
#     已经签到 / 已經簽到 / 您已簽到 / 您已签到
#   匹配到任一 → 返 (True, diag with matched_marker)
#   没匹配 → 看是否登录态(logout / 登出 / userdetails)→ 真未签
# 兜底成功 → 转 JmAlreadySignedToday → 平台语义 success
```

## 经验教训

| 教训 | 适用场景 |
|---|---|
| 同 IP 同天 CF Turnstile 第二次必升级 | 任何反复测试 selenium 反爬脚本时,**不要一天跑 2 次**;开发测试时换 mock IP / Tailnet exit-node |
| 用户认知 ≠ 真相("我今早签了" 实际 sign 接口返空假成功) | PM 不能信用户口头汇报,**必须看日志原文**(grep 关键字) |
| 笼统 `Exception as e: print` 是反模式 | 任何远程 HTTP 调用都应**结构化分类异常**:网络层 / HTTP 层 / 业务层各自 raise 不同类 |
| `json.loads(text)` 前必须校验 status_code / Content-Type / 非空 | 不然 `JSONDecodeError` 笼统消息无法定位 |
| 反爬"静默返空" 比 "明确 403" 更难诊断 | 任何抓数据的脚本应有**兜底数据源**(此例:GET 首页 marker) |
