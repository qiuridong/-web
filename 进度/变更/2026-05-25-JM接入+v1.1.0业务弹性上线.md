## 学到的 5 个关键知识

### 1. CF 信任分模型(实测铁证)

同 IP 同天 selenium 调用次数 → CF 升级难度:
- 第 1 次:大概率给 Turnstile 按钮 → UC 模式能过
- 第 2-3 次:CF 升级到 invisible challenge → 不给按钮 → 任何 SeleniumBase 模拟点击都过不去
- 24h 后 IP 信任分自然恢复

**对应措施**:`session 复用`(同次执行内重试不再过 CF)+ `cf_clearance_ttl_sec` 自管理(避免过早过 CF)

### 2. JM 18comic 反爬层(实测)

- **L1 Cloudflare Turnstile**(JS Challenge,需真 Chrome 或长期 cf_clearance 复用)
- **L2 业务层反爬**:sign 接口可能对自动化流量返**空 body**(5-23 事件)或 `{"msg":""}` anonymous
- **L3 cookie ↔ server IP 强绑定**(sticky cookie 揭示)

### 3. NexusPHP 登录令牌机制

JM 18comic 用 PHP `remember` cookie 做长期登录令牌(不是 PHPSESSID),含 username + MD5 password + SHA1 签名。这与 ptfans(同 NexusPHP)的 `c_secure_pass` 类似但具体实现不同。

### 4. 桌面 vs 手机签到 endpoint 可能不同

桌面网页签到后**按钮消失无法重抓** → HAR 抓不到真实 sign 请求样本。手机端有独立签到流程(未调研),**可能用不同 endpoint 且反爬更宽松**(后续优化方向)。

### 5. 错误诊断结构化的真实价值

5-23 事件 host 原版只笼统打印 "请求异常: Expecting value: line 1 column 1 (char 0)" → 排查超痛苦。v1.1.0 现在每次失败 `RunResult.data` 含:
```json
{
  "error_class": "JmSignEndpointError",
  "endpoint": "POST /ajax/user_daily_sign",
  "status_code": 200,
  "content_type": "text/html; charset=UTF-8",
  "content_length": 0,
  "body_preview": "<empty>",
  "body_len": 0,
  "elapsed_ms": 156
}
```
→ 前端 /runs 详情页 5 秒定位。
